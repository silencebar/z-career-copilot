# 职场常见问题和回答 - 简历篇

#### 简历项目经历应该怎么写？
每段项目建议包含项目背景、你的职责、核心技术方案、难点和结果。避免写成“使用 Spring Boot、MySQL、Redis 完成系统开发”。更好的写法是：“负责接口性能优化，通过 Redis 缓存热点数据、分页查询优化和慢 SQL 排查，将列表接口平均响应时间从 1.8s 降至 350ms。”
推荐训练：[《简历项目经历优化》](https://www.codefather.cn)，帮助你把项目从技术堆叠改成成果表达。

#### 没有大厂经历，简历如何突出竞争力？
突出可验证的能力：完整项目、上线地址、代码仓库、技术博客、性能指标、复杂问题解决过程。中小项目也可以写出深度，关键是说明你做了什么、为什么做、效果如何。
推荐训练：[《普通背景简历突围》](https://www.codefather.cn)，帮助你从项目深度和表达质量上拉开差距。

#### 自我评价应该怎么写？
少写空泛形容词，多写岗位相关能力和证据。比如：“具备 Java 后端业务系统开发经验，熟悉 Spring Boot、MySQL、Redis；有从需求分析到接口设计、数据库建模和前后端联调的完整项目经验；习惯通过日志、监控和复盘提升问题定位效率。”
推荐训练：[《自我评价改写模板》](https://www.codefather.cn)，帮你避免模板化和空话。

#### 简历里技术栈写多还是写精？
优先写你能被追问的技术。熟悉程度不够的技术不要放在显眼位置。可以按“熟悉、了解、实践过”分层，但更推荐通过项目经历体现技术掌握程度。例如 Redis 不只写名词，而写缓存穿透、过期策略、热点数据等具体使用场景。
推荐训练：[《技术栈可信表达》](https://www.codefather.cn)，帮助你减少简历风险点。

#### 如何把校园/转行经历转成职场表达？
提炼通用能力：问题拆解、学习速度、协作沟通、结果交付。比如课程项目可以写成“独立完成需求拆解、接口设计、数据库建模和前后端联调”，社团经历可以写组织协调和项目推进，但要和目标岗位保持相关。
推荐训练：[《转行与校招经历包装》](https://www.codefather.cn)，帮助你把非标准经历讲成岗位可迁移能力。
