# 职场常见问题和回答 - 面试篇

#### 面试时如何清晰介绍项目经历？
先用一句话说明项目定位和业务价值，再按“背景-目标-职责-方案-结果-复盘”展开。重点讲你负责的模块、遇到的难点、为什么这样设计、最后带来了什么效果。避免只罗列技术名词，要把技术选择和业务收益连起来。例如：“我负责登录鉴权模块，采用 JWT + Redis 维护登录态，解决了多端访问下会话校验和权限控制的问题。”
推荐训练：[《项目经历表达训练》](https://www.codefather.cn)，通过 STAR 框架和追问清单帮助你把项目讲得更像真实业务经验。

#### 被问到不会的问题怎么办？
先承认边界，不要硬编答案。可以说：“这个点我没有完整实践过，但我理解它主要解决的是……如果让我排查，我会先从……入手。”然后补充你已知的相关知识、推理过程和后续学习计划。面试官通常更看重思考方式、诚实度和学习能力。
推荐训练：[《面试追问应对课》](https://www.codefather.cn)，帮助你把不会的问题转化成可展示思路的回答。

#### 如何回答“你的优势是什么”？
不要只说“学习能力强、责任心强”。建议结合岗位要求给出 2 到 3 个具体优势，每个优势配一个证据。例如：“我的优势是能把需求拆成可落地方案。在 XX 项目中，我把用户登录流程拆成参数校验、鉴权、异常处理和日志追踪四块，减少了线上排查时间。”
推荐训练：[《面试自我介绍与优势表达》](https://www.codefather.cn)，帮你准备更有证据感的个人卖点。

#### 面试前一天应该准备什么？
梳理岗位 JD，标出高频技能点；准备 1 分钟和 3 分钟两个版本的自我介绍；复盘 2 个核心项目；整理常见八股题；准备反问问题，如团队业务、技术栈、培养机制。最后检查路线、简历、作品链接和电脑设备。
推荐训练：[《面试前冲刺清单》](https://www.codefather.cn)，用清单方式降低临场遗漏。

#### 如何向面试官解释离职或转岗原因？
表达要积极、客观、面向成长。避免抱怨前公司或同事。可以说：“上一段经历让我积累了 XX 能力，但我希望在更贴近后端工程化/业务系统建设的岗位中继续成长，所以选择寻找新的机会。”再自然连接到目标岗位。
